API Reference

ProxyGate exposes a REST API on the admin port for managing routes, certificates, and DNS records. All endpoints accept and return JSON.

Base URL: http://127.0.0.1:9090/api

Configuration

Read and update the ProxyGate server configuration.

GET /api/config Get current configuration

Response

{
  "server": { "http_port": 80, "https_port": 443, "admin_port": 9090, "admin_host": "127.0.0.1", "allowed_networks": ["127.0.0.0/8"] },
  "routes": [...],
  "godaddy": { "api_key": "********" },
  "acme": { "email": "admin@example.com", "use_staging": true }
}

Note: GoDaddy API secret is redacted in the response.

PUT /api/config/server Update server configuration

Request Body

{
  "http_port": 80,
  "https_port": 443,
  "admin_port": 9090,
  "admin_host": "127.0.0.1",
  "allowed_networks": ["127.0.0.0/8", "192.168.1.0/24"]
}

Proxy Routes

Manage reverse proxy routes. Changes take effect immediately without restart.

GET /api/routes List all routes
POST /api/routes Create a new route

Request Body

{
  "name": "My App",
  "domain": "app.example.com",
  "target_host": "127.0.0.1",
  "target_port": 3000,
  "tls_enabled": false,
  "enabled": true
}
GET /api/routes/{id} Get a specific route
PUT /api/routes/{id} Update a route
DEL /api/routes/{id} Delete a route
GET /api/status Proxy status and health

TLS Certificates (ACME)

Request and manage Let's Encrypt certificates using DNS-01 challenge.

POST /api/acme/request Initiate certificate request

Request Body

{ "domain": "app.example.com" }

Response

{
  "domain": "app.example.com",
  "status": "pending",
  "record_name": "_acme-challenge.app.example.com",
  "record_value": "dGVzdC12YWx1ZQ..."
}

Create the returned TXT record at your DNS provider, then call the complete endpoint.

POST /api/acme/complete Verify challenge and fetch certificate

Request Body

{
  "domain": "app.example.com",
  "auto_dns": true
}

Set auto_dns: true to automatically create the TXT record via GoDaddy before verification.

GET /api/acme/challenges List active ACME challenges
PUT /api/acme/config Update ACME settings

GoDaddy DNS

Manage DNS records through the GoDaddy API. Supports A, CNAME, TXT, and other record types.

PUT /api/godaddy/config Update GoDaddy API credentials

Request Body

{
  "api_key": "your-key",
  "api_secret": "your-secret",
  "base_url": "https://api.godaddy.com"
}
POST /api/godaddy/verify Test API connection
GET /api/godaddy/domains List all domains
GET /api/godaddy/domains/{domain}/records List DNS records
POST /api/godaddy/domains/{domain}/records Create a DNS record

Request Body

{
  "type": "A",
  "name": "app",
  "data": "192.168.1.100",
  "ttl": 600
}
DEL /api/godaddy/domains/{domain}/records/{type}/{name} Delete a DNS record
POST /api/godaddy/auto-dns Auto-create DNS for a proxy route

Request Body

{
  "domain": "example.com",
  "subdomain": "app",
  "type": "A",
  "value": "192.168.1.100"
}

Real-time Events

Connect via WebSocket for live updates when configuration, routes, or certificates change.

GET /ws WebSocket connection

Event Types

config_changed   — Server configuration was updated
route_added      — A new proxy route was created
route_updated    — An existing route was modified
route_deleted    — A route was removed
cert_obtained    — A TLS certificate was issued
acme_challenge   — An ACME challenge was created
dns_record_created — A DNS record was created
auto_dns_created — Auto-DNS record was created

Message Format

{
  "type": "route_added",
  "payload": { ... }
}
